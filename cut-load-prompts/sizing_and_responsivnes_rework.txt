How article size is currently determined
The overall feature page card height is forced by .newspaper-page--tall .glass { min-height: calc(200vh - 80px); } in src/styles/global.css.
Under @media (max-width: 1080px), that drops to min-height: calc(100vh - 80px).
Inside that fixed-height card, .feature-article-pad and .timeline-section use flex with flex: 1, so the timeline gets “remaining height”.
Timeline cards then size from .timeline-col with height: 100% + aspect-ratio.
So yes: card width should derive from height. But then…

Why your ratio keeps changing
At the bottom of src/styles/global.css, these media rules override the base value:

@media (orientation: landscape) { .timeline-col { --timeline-col-ratio: 3 / 2; } }
@media (orientation: portrait) { .timeline-col { --timeline-col-ratio: 3 / 5; } }
That means your 13 / 20 is ignored in landscape.

Root layout issue (forced horizontal timeline)
These rules force horizontal behavior regardless of available vertical space:

.timeline-scroll-wrap { overflow-x: auto; overflow-y: hidden; }
.timeline-scroll { display: flex; min-width: max-content; }
.timeline-col { flex-shrink: 0; }
So the timeline will always become a sideways strip.

What has to change
You need a layout mode switch, not just ratio tweaks:

Remove or neutralize orientation-based ratio overrides (or move your desired ratio after them).
Add two timeline modes:
horizontal (current behavior)
vertical (stacked cards, no horizontal scroll)
Switch mode by space, not device orientation:
Prefer a container query (if you want pure CSS), or
Use JS to measure timeline area and toggle a class when height is generous vs width.
A good rule of thumb:

If timeline container is wide/short -> horizontal.
If container is narrow/tall -> vertical.
If you want, I can outline the exact CSS for both modes and the smallest JS/class toggle needed (or a pure container-query version).
